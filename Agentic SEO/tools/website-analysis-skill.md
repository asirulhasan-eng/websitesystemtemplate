## 1. Overview

When a client website is imported, three things need to happen:

1. **Mechanical** -- Replace `{{TOKENS}}` and structural values (paths, IDs, handles). The CLI scripts (`analyze-website.ps1`, `inject-skills.ps1`) handle this.

2. **Analytical** -- Understand what the business does, who they serve, how they talk, what their industry cares about. **Only the AI can do this.**

3. **Creative** -- Rewrite voice guides, angle formulas, humor examples, subreddit suggestions, and research queries so they match the client's world. **Only the AI can do this.**

This skill handles all three layers. The CLI scripts are tools called during Phase 4. Phases 1-3 and 5-6 are pure AI work.

### Target Files

These are the files you will customize:

```
Agentic SEO/tools/
  blog-production-skill.md           <-- Blog writing workflow + voice guide
  stats-blog-production-skill.md     <-- Statistics post workflow
  SERVICE-PAGE-PRODUCTION-SKILL.md   <-- Service page workflow + voice guide
  scaffold-blog.ps1                  <-- Blog scaffolding script
```

### Config Files You Will Fill

```
site.config.json                     <-- Client identity + tokens
website-profile.json                 <-- Generated by analyze-website.ps1
```

## 2. Phase 1: Read & Understand the Website

> **CRITICAL: Do NOT skip this phase. Do NOT rush to run scripts.** The entire point of this skill is that you THINK about the website before touching anything. Read everything first.

### Step 1.1 -- Read the Home Page

Open `Website/index.html` and read the FULL content. Extract:

```
BUSINESS IDENTITY:
  - What is the business name?
  - What do they actually do? (not what they say they do -- what SERVICE do they provide?)
  - Who is their customer? (homeowners? businesses? contractors? patients? restaurants?)
  - What geographic area do they serve?
  - What is their primary value proposition? (fast? cheap? reliable? specialized?)
  - What makes them different from competitors?

BRAND SIGNALS:
  - What is the tone of the copy? (formal? casual? technical? friendly? authoritative?)
  - Do they use first person ("we") or third person ("the company")?
  - Is the language simple or complex?
  - Do they use industry jargon or plain English?
  - Are there testimonials? What do customers praise?
  - What CTAs do they use? ("Call now"? "Get a quote"? "Schedule"? "Learn more"?)

VISUAL IDENTITY:
  - What is the primary brand color? (from header, buttons, accents)
  - What is the background style? (light? dark? gradient?)
  - What font style? (serif? sans-serif? modern? traditional?)
  - Logo style? (text-based? icon? monogram?)
```

### Step 1.2 -- Read ALL Blog Posts

For EVERY blog post in `Website/blog/`:

1. Open and read the full content (not just the title)
2. Note the writing style -- is it first-person? Educational? Conversational?
3. What topics do they cover?
4. How long are posts? (rough word count)
5. Do they use infographics, data tables, or just text?
6. What is the internal linking pattern?
7. What CTA appears at the bottom?
8. What categories/tags do they use?

**Build a mental model of:**
- What the blog "sounds like" when you read it
- What topics they care about
- How they structure arguments
- Whether they cite data or write opinion pieces
- How they refer to their customers ("homeowners"? "business owners"? "clients"? "patients"?)

### Step 1.3 -- Read ALL Service Pages

For EVERY page in `Website/pages/` (or `Website/services/`):

1. Read the full content
2. What services do they offer?
3. How do they describe each service?
4. Do they show pricing?
5. What objections do they address?
6. What trust signals do they use? (years in business, certifications, guarantees)
7. What is the conversion path? (call? form? quote request?)

### Step 1.4 -- Read the CSS

Open `Website/css/main.css` and extract:

```
CSS CUSTOM PROPERTIES (--variables):
  List every CSS variable and its value.
  Identify which are brand colors vs. utility colors.

KEY COLOR USAGE:
  - Header background color
  - Button primary color
  - Link color
  - Footer background
  - Text color
  - Accent/highlight color
```

### Step 1.5 -- Read the JavaScript

Scan `Website/js/` files for:
- Analytics IDs (GA4, GTM, Clarity, Facebook Pixel, etc.)
- Third-party integrations (chat widgets, form handlers, booking tools)
- Social media embed patterns

### Step 1.6 -- Read the Footer & Navigation

From any page, extract:
- All navigation links (main menu items)
- Footer content (phone number, address, service area, hours)
- Social media links (every platform)
- Legal pages (privacy, terms)
- Schema markup (open the source, look for `application/ld+json`)

## 3. Phase 2: Build the Client Profile

After reading everything, answer these questions IN WRITING. Do not skip any.
Write your answers into a scratch file or artifact before proceeding.

### 2.1 -- Business Identity (Required)

```
NICHE:          [Single word or hyphenated phrase -- the industry]
                Examples: plumbing, roofing, hvac, dental, landscaping,
                pest-control, electrical, auto-repair, real-estate

AUDIENCE:       [Who is the customer, in one word, plural]
                Examples: homeowners, business-owners, contractors, patients,
                restaurant-owners, property-managers, car-owners

DOMAIN:         [The website's domain, no protocol]
                Example: acmeplumbing.com

SITE_NAME:      [The business/brand name as it appears on the site]
                Example: Acme Plumbing

SITE_DESCRIPTION: [One sentence describing what they do]
                Example: Professional plumbing services for homeowners in Dallas-Fort Worth.

OWNER_NAME:     [If visible on the site, otherwise leave as placeholder]
```

### 2.2 -- Voice Analysis (Required)

Answer these based on what you READ, not what you assume:

```
1. REGISTER:    [Formal / Semi-formal / Casual / Very casual]
2. PERSON:      [First person "we" / Third person "the company" / Mixed]
3. JARGON:      [Heavy industry jargon / Some jargon / Plain English]
4. SENTENCE LENGTH: [Short and punchy / Medium / Long and detailed]
5. EMOTIONAL TONE:  [Reassuring / Authoritative / Friendly / Urgent / Clinical]
6. HUMOR:       [None / Rare and dry / Occasional / Frequent]
7. CUSTOMER REFERENCE: [What word do they use for customers?]
                       homeowners / business owners / clients / customers / patients / etc.
8. PAIN POINTS: [What fears/problems does the copy address?]
                Cost anxiety / quality concerns / trust issues / urgency / complexity / etc.
9. TRUST SIGNALS: [How do they build credibility?]
                  Years in business / certifications / guarantees / reviews / case studies
10. CTA STYLE:  [Soft "learn more" / Medium "get a quote" / Hard "call now"]
```

### 2.3 -- Brand Voice Statement (Required)

Write a 2-3 sentence brand voice description based on your analysis. This goes into `site.config.json` as `business.brand_voice`.

**Template:**
"[Register], [emotional tone]. Speaks [directly to / about] the [audience] as a [relationship metaphor]. [Key characteristic of the writing style]."

**Examples:**
- "Expert, straightforward, ROI-focused. Speaks directly to the business owner and ties every recommendation to tangible outcomes (calls, booked jobs, revenue)."
- "Warm, reassuring, patient-focused. Speaks to anxious patients like a trusted family doctor explaining options. Uses simple language, avoids medical jargon, emphasizes comfort and outcomes."
- "Confident, no-nonsense, trade-aware. Speaks to homeowners like a trusted neighbor who happens to be a licensed contractor. Short sentences, honest pricing talk, zero marketing fluff."

### 2.4 -- Industry Context (Required)

```
INDUSTRY PAIN POINTS (top 5):
  1. [What keeps the business owner up at night?]
  2. [What do customers complain about in this industry?]
  3. [What are the common scams/bad actors?]
  4. [What seasonal/cyclical pressures exist?]
  5. [What is the competitive landscape like?]

RELEVANT SUBREDDITS (5-8):
  r/[niche-specific]
  r/[customer-community]
  r/[industry-professionals]
  r/[local-community]
  r/[related-topic]

RELEVANT YOUTUBE CHANNELS (2-4):
  [Channels that cover this niche -- for transcript mining]

COMPARISON ANGLES (what are the key "X vs Y" topics in this niche?):
  [Service A] vs [Service B]
  [DIY] vs [Professional]
  [This approach] vs [That approach]

COMMON SEARCH PATTERNS (how do customers search for this?):
  "[service] near me"
  "[service] cost"
  "how much does [service] cost"
  "best [service] [city]"
  "[service] vs [alternative]"
```

### 2.5 -- Content Structure Analysis (Required)

```
BLOG POST STRUCTURE:
  - Average word count: [from reading the posts]
  - Typical sections: [list the H2 pattern you observed]
  - Uses infographics? [yes/no, how many per post?]
  - Uses data tables? [yes/no]
  - FAQ section? [yes/no, schema?]
  - TLDR/summary box? [yes/no]
  - Related posts section? [yes/no, how many?]
  - CTA placement: [top only / bottom only / top+bottom / inline+bottom]

SERVICE PAGE STRUCTURE:
  - Typical sections: [hero, overview, how-it-works, pricing, FAQ, CTA, etc.]
  - Shows pricing? [yes/no/ranges]
  - Process steps? [yes/no, how many?]
  - Testimonials? [yes/no]
  - Schema types used: [Service, LocalBusiness, FAQ, etc.]
```

## 4. Phase 3: Fill site.config.json

Using your Phase 2 analysis, fill `site.config.json` with the client's real values.

```powershell
# Open and edit site.config.json
# Replace ALL placeholder values with real ones from your analysis
```

**Fields to fill:**

| Field | Source |
|---|---|
| `domain` | From canonical URL or og:url |
| `site_name` | From og:site_name or site header |
| `site_description` | Write from your understanding (1 sentence) |
| `owner_name` | From About page, footer, or schema |
| `business.niche` | Your Phase 2.1 NICHE answer |
| `business.audience` | Your Phase 2.1 AUDIENCE answer |
| `business.brand_voice` | Your Phase 2.3 brand voice statement |
| `email.admin` | From contact page or schema |
| `timezone` | From business location (IANA format) |
| `timezone_abbr` | Short timezone (ET, CT, PT, etc.) |
| `cloudflare.project_name` | Derived from domain (lowercase, hyphens) |

> **IMPORTANT:** The `niche` field should be a SINGLE WORD or hyphenated phrase (e.g., "plumbing", "pest-control", "auto-repair"). It gets used in folder names, CSS class prefixes, and URL patterns. Keep it simple.

> **IMPORTANT:** The `audience` field should be a PLURAL NOUN (e.g., "homeowners", "roofers", "dentists"). It gets used in phrases like "for {{AUDIENCE}}" and "The Bottom Line for Busy {{AUDIENCE}}".

## 6. Phase 5: Deep Customization (AI-Only Work)

> **THIS IS THE MOST IMPORTANT PHASE.** The CLI scripts did the mechanical work. Now you must do the work that ONLY an AI with understanding can do. Do NOT skip any section.

After the scripts run, open each skill file and customize these sections using your Phase 2 analysis.

### 5.1 -- Rewrite the Voice Guide (blog-production-skill.md)

Open `Agentic SEO/tools/blog-production-skill.md` and find the "Voice Guide" section.

**What to change:**

1. **Opening description** -- "Write as a tradesman-turned-owner speaking to another {{NICHE}} business owner."
   - Rewrite this to match the actual relationship dynamic.
   - A dental SEO agency doesn't talk like a tradesman. A restaurant SEO agency doesn't talk like a contractor.
   - Match the voice register you identified in Phase 2.2.

2. **"Primary reader" paragraph** -- "They carry payroll, insurance, dispatch, trucks..."
   - Rewrite with the ACTUAL pressures of this industry.
   - A dentist carries patient liability, staff scheduling, equipment costs, insurance billing.
   - A restaurant owner carries food costs, staff turnover, health inspections, seasonal demand.
   - Use the REAL pain points from your Phase 2.4 analysis.

3. **"Voice pillars"** -- "Plainspoken. Owner-pressure aware. Translate SEO to business outcomes."
   - Keep the structure (3 pillars) but rewrite the examples.
   - "quiet phones, wrong-area calls, competitors ranking above them" -- replace with THIS industry's version.

4. **"Use language like"** list -- "phone doesn't quit ringing, keep the trucks moving..."
   - Replace EVERY phrase with industry-appropriate equivalents.
   - For a dental practice: "chairs stay full, new patient calls, hygiene recall rate, case acceptance"
   - For a restaurant: "tables stay full, reservation book, online orders, review management"
   - For a roofer: "leads coming in, estimates booked, storm season, insurance jobs"
   - **Pull these from the actual website copy you read in Phase 1.**

5. **"Avoid language like"** list
   - Keep the same general principle (avoid hollow marketing speak)
   - But adjust if the industry uses some of these terms legitimately

6. **Humor section** -- "Bad SEO is like a slow drain..."
   - Rewrite the allowed example using an industry-appropriate metaphor.
   - Keep the humor rules (light, dry, practical) but change the trade reference.

7. **Opinion phrases** -- "The phone tells the truth, that is not growth that is noise..."
   - Rewrite with industry-specific outcome references.

8. **"Before writing, check"** list
   - Adapt question 5: "Does it sound like an owner talking near a truck" -> match the industry setting.

### 5.2 -- Rewrite the Voice Guide (SERVICE-PAGE-PRODUCTION-SKILL.md)

Same process for the service page voice guide. This one has different framing:
- "Write as a tradesman-turned-owner speaking to another homeowner"
- Adapt for the actual service relationship (e.g., doctor-to-patient, chef-to-diner, contractor-to-homeowner)

### 5.3 -- Update Research Suggestions

In ALL THREE skill files, find the research sections and update:

1. **Reddit subreddit suggestions** -- Replace `r/{{NICHE}}, r/HomeImprovement, r/smallbusiness` with:
   - The actual subreddits from your Phase 2.4 analysis
   - At least 5 relevant subreddits

2. **Search query examples** -- Replace generic queries with niche-specific ones:
   - `"[topic] for {{AUDIENCE}}"` is fine (it uses tokens)
   - But contextual examples like `"site:reddit.com [TOPIC] contractor"` should use the right profession

3. **YouTube transcript research** -- If the niche has relevant YouTube channels, mention them as research targets

### 5.4 -- Update the Brand Palette

In `blog-production-skill.md` and `stats-blog-production-skill.md`, find the **Brand palette** section:

```
- Navy bg: #0F2A44
- Primary accent: #176B9A
- Alert/negative: #E94743
- Success/positive: #74C9A7
- Highlight: #F4D77A
- Text on dark: #FAFAF7
```

**Replace these with the client's actual brand colors** from your Phase 1.4 CSS analysis. If the client uses a light theme, adapt the infographic palette accordingly. The palette should feel like it belongs on the client's website.

### 5.5 -- Update Project Structure References

In all three skill files, verify the "Project Structure" code blocks match the actual imported website's directory layout. The CLI scripts handle the path tokens, but the TREE DIAGRAM itself may need adjustment if the client uses different folder names (e.g., `services/` instead of `pages/`, or `img/` instead of `assets/images/`).

### 5.6 -- Update the Blog Angle Formula (blog-production-skill.md)

Find Step 2.1b "Blog Angle Formula" and rewrite:
- The competitive framing ("quiet phones, wrong-area calls, competitors above you")
- The pain-point progression
- The outcome references

Use the ACTUAL pain points and outcomes from your Phase 2.4 industry analysis.

### 5.7 -- Update Schema/Social Examples

In all skill files and in `scaffold-blog.ps1`, verify:
- Social media `sameAs` URLs are correct for this client
- Schema `author`/`publisher` names match
- Twitter handle is correct
- Any hardcoded example URLs match the client's domain

### 5.8 -- Update the Service Page Angle Formula (SERVICE-PAGE-PRODUCTION-SKILL.md)

Find "Service Angle Formula" (section 2.1b) and adapt:
1. "Start with the owner's fear" -- what DOES this industry's customer fear?
2. "Explain why DIY fails" -- is DIY even relevant? (dental = no, roofing = yes)
3. "Show what a professional does" -- what differentiates in THIS industry?
4. "Connect to outcomes" -- what outcome does this customer want?
5. "Make the ask clear" -- what's the conversion action? (call? book? schedule?)

## 7. Phase 6: Verification

### 6.1 -- Token Check

Run this to find any remaining unfilled tokens:

```powershell
Select-String -Path "Agentic SEO\tools\*-skill.md","Agentic SEO\tools\SERVICE-PAGE*.md","Agentic SEO\tools\scaffold-blog.ps1" -Pattern "\{\{" | Format-Table Path, LineNumber, Line -AutoSize
```

Every remaining `{{TOKEN}}` is a bug. Fix it.

### 6.2 -- Voice Consistency Check

Re-read the voice guide sections you wrote. Ask yourself:
- [ ] If I read this voice guide cold, could I write content that sounds like the client's existing blog posts?
- [ ] Are the "use language like" phrases actually from this industry?
- [ ] Are the "avoid language like" phrases things this industry would actually misuse?
- [ ] Does the humor example make sense in this industry's world?
- [ ] Would the client recognize themselves in this voice guide?

### 6.3 -- Research Relevance Check

- [ ] Are the suggested subreddits real and active communities for this niche?
- [ ] Do the example search queries match how real customers search for this service?
- [ ] Are the comparison angles ("X vs Y") real decisions customers face?

### 6.4 -- Structural Check

- [ ] Do the project structure diagrams match the actual Website/ directory?
- [ ] Does the scaffold script reference a real existing blog post as source?
- [ ] Are the CSS file references correct (main.css, blog.css, pages.css)?
- [ ] Are analytics IDs filled (or marked as TODO if not present)?

### 6.5 -- Final Read-Through

Read each skill file from top to bottom. At no point should you feel like you're reading about a different business. Every example, every reference, every query should feel like it belongs to THIS client.
